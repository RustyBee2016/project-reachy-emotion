# Thumbnail Generation Architecture

## Overview

Thumbnails are automatically generated by a background service that watches for new videos in the `/videos/temp` directory.

## Architecture

### Components

1. **ThumbnailGenerator** (`shared/utils/thumbnail_generator.py`)
   - Utility class that wraps FFmpeg for thumbnail extraction
   - Extracts a frame at 1 second into the video
   - Generates high-quality JPEG thumbnails
   - Supports multiple video formats (mp4, avi, mov, mkv, webm, etc.)

2. **ThumbnailWatcherService** (`apps/api/app/services/thumbnail_watcher.py`)
   - Background service that runs continuously
   - Polls configured directories every 5 seconds (configurable)
   - Automatically generates thumbnails for new videos
   - Tracks processed videos to avoid regeneration
   - Handles errors gracefully without crashing

3. **Integration** (`apps/api/app/main.py`)
   - Service starts automatically when Media Mover API starts
   - Stops gracefully on shutdown
   - Watches the `temp` split by default

## Workflow

```
1. Video arrives in /videos/temp/
   ↓
2. ThumbnailWatcherService detects new video (within 5 seconds)
   ↓
3. ThumbnailGenerator extracts frame using FFmpeg
   ↓
4. Thumbnail saved to /videos/thumbs/{video_id}.jpg
   ↓
5. Thumbnail endpoint /api/v1/media/{video_id}/thumb returns URL
```

## Configuration

### Environment Variables

- `REACHY_THUMBS_DIR`: Directory name for thumbnails (default: `thumbs`)
- `REACHY_VIDEOS_ROOT`: Root directory for all video storage

### Service Configuration

The watcher service can be configured in `apps/api/app/main.py`:

```python
_thumbnail_watcher = ThumbnailWatcherService(
    videos_root=config.videos_root,
    watch_splits=["temp"],  # Directories to watch
    poll_interval=5.0,      # Seconds between scans
)
```

## n8n Ingest Agent Updates

### Previous Approach (DEPRECATED)

The old workflow attempted to call `/api/media/pull` with `compute_thumb=true`, but this endpoint was never implemented.

### Current Approach (AUTOMATIC)

**No changes needed to n8n workflows!**

Thumbnails are now generated automatically by the background service. The n8n Ingest Agent simply needs to:

1. Download/move video to `/videos/temp/`
2. Insert video metadata into database
3. Emit completion event

The thumbnail will be generated automatically within 5 seconds.

### Workflow Simplification

You can **remove** these parameters from the media.pull call (if it exists):
- `compute_thumb` - No longer needed
- `ffprobe` - Should be handled separately if needed

The background service handles thumbnail generation independently.

## API Endpoints

### Get Thumbnail URL

```bash
GET /api/v1/media/{video_id}/thumb
```

**Response (Success):**
```json
{
  "status": "success",
  "data": {
    "video_id": "luma_1",
    "thumbnail_url": "http://10.0.4.130:8082/thumbs/luma_1.jpg"
  },
  "meta": {
    "correlation_id": "...",
    "timestamp": "2025-11-19T08:40:08.256293Z",
    "version": "v1"
  }
}
```

**Response (Not Found - Thumbnail Not Yet Generated):**
```json
{
  "detail": {
    "error": "not_found",
    "message": "Thumbnail not found for video: luma_1"
  }
}
```

### Handling Race Conditions

If you query the thumbnail endpoint immediately after video ingestion, the thumbnail might not exist yet (404). Options:

1. **Wait and Retry**: Poll the endpoint with exponential backoff
2. **Event-Driven**: Wait for a thumbnail.generated event (future enhancement)
3. **Graceful Degradation**: Show placeholder image if thumbnail not available

## Troubleshooting

### Thumbnails Not Being Generated

1. **Check FFmpeg is installed:**
   ```bash
   ffmpeg -version
   ```

2. **Check service logs:**
   ```bash
   # Look for "Thumbnail watcher service started"
   # Look for "Generated thumbnail for: {video_id}"
   ```

3. **Check directory permissions:**
   ```bash
   ls -ld /media/rusty_admin/project_data/reachy_emotion/videos/thumbs/
   # Should be writable by the API user
   ```

4. **Manually test thumbnail generation:**
   ```bash
   ffmpeg -ss 00:00:01 -i /path/to/video.mp4 -vframes 1 -q:v 2 /tmp/test_thumb.jpg
   ```

### Service Not Starting

Check logs for errors like:
- `FFmpeg not found` - Install FFmpeg
- `Permission denied` - Fix directory permissions
- `Configuration validation failed` - Check environment variables

## Performance Considerations

- **Poll Interval**: Default 5 seconds balances responsiveness vs. CPU usage
- **Concurrent Processing**: Thumbnails are generated sequentially to avoid overwhelming FFmpeg
- **Resource Usage**: Each thumbnail generation uses ~50-200MB RAM briefly
- **Disk Space**: Thumbnails are ~10-50KB each (JPEG quality 2)

## Future Enhancements

1. **Event Emission**: Emit `thumbnail.generated` events for reactive workflows
2. **Batch Processing**: Process multiple videos in parallel with concurrency limits
3. **Custom Timestamps**: Allow extracting thumbnails from different timestamps
4. **Multiple Thumbnails**: Generate thumbnail strips or multiple preview images
5. **Cleanup Service**: Remove orphaned thumbnails when videos are deleted
6. **Health Endpoint**: Add `/health/thumbnails` to monitor service status

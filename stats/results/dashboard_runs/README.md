# Dashboard Run Results (Variant 1 + Variant 2)

`06_Dashboard.py` reads run payloads from this folder layout:

`stats/results/dashboard_runs/<variant>/<run_type>/<run_id>.json`

Valid `<variant>` values:
- `variant_1`
- `variant_2`

Valid `<run_type>` values:
- `training`
- `validation`
- `test`

Run IDs used by the six dropdown selections:
- `Variant 1 Training Run` -> `variant_1/training/run_0001.json`
- `Variant 1 Validation Run` -> `variant_1/validation/run_0002.json`
- `Variant 1 Test Run` -> `variant_1/test/run_0003.json`
- `Variant 2 Training Run` -> `variant_2/training/run_1001.json`
- `Variant 2 Validation Run` -> `variant_2/validation/run_1002.json`
- `Variant 2 Test Run` -> `variant_2/test/run_1003.json`

If an exact run ID file is missing, the dashboard will automatically load the
most recent JSON file in the corresponding `<variant>/<run_type>/` folder.

Pipeline artifact storage is variant + run-type partitioned:

`stats/results/<variant>/<run_type>/<run_id>/predictions.npz`

`stats/results/<variant>/<run_type>/<run_id>/gate_a.json`

These are generated by `trainer/run_efficientnet_pipeline.py`, which also
writes this dashboard JSON payload.

Minimum JSON payload shape:

```json
{
  "run_id": "run_0001",
  "run_type": "training",
  "model_variant": "variant_1",
  "gate_a_metrics": {
    "accuracy": 0.92,
    "precision_macro": 0.91,
    "recall_macro": 0.90,
    "f1_macro": 0.905,
    "balanced_accuracy": 0.90,
    "f1_happy": 0.91,
    "f1_sad": 0.90,
    "ece": 0.07,
    "brier": 0.10,
    "mce": 0.15,
    "confusion_matrix": [[50, 5], [6, 49]]
  },
  "gate_a_gates": {
    "macro_f1": true,
    "balanced_accuracy": true,
    "per_class_f1": true,
    "ece": true,
    "brier": true,
    "mce": true
  }
}
```

If no run JSON file exists for that variant/run_type, or a file is invalid,
the dashboard falls back to an in-code placeholder payload and shows a warning.

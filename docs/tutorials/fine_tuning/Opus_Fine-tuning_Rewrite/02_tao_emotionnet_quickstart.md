# Tutorial 02: TAO EmotionNet Quick-Start

## Overview

This tutorial consolidates the existing TAO-based training pipeline for
EmotionNet. It covers the complete workflow: dataset preparation, TAO
fine-tuning, quality gate validation, TensorRT export, and DeepStream
deployment. This is **Path A** from the [overview](01_overview.md).

> **Prerequisite**: Docker must be running with the TAO containers available.
> See `trainer/tao/docker-compose-tao.yml` for container definitions.

---

## Architecture

EmotionNet uses a **ResNet-18** backbone pretrained on ImageNet. The project
supports two configurations:

| Config | File | Classes | Epochs | Batch Size | Gate A (F1) |
|--------|------|---------|--------|------------|-------------|
| 2-class | `trainer/tao/specs/emotionnet_2cls.yaml` | happy, sad | 50 | 32 | ≥ 0.84 |
| 6-class | `trainer/tao/specs/emotionnet_6cls.yaml` | happy, sad, angry, neutral, surprise, fearful | 80 | 24 | ≥ 0.75 |

Both configs use:
- Input resolution: 224 × 224 × 3
- Optimizer: Adam with cosine LR schedule and warmup
- Label smoothing: 0.1 (2-cls) / 0.15 (6-cls)
- Mixed precision (FP16)
- Class-balanced weighting
- Early stopping on `val_f1_macro`

## Step 1: Prepare the Dataset

The project uses synthetic videos from Luma AI. TAO operates on **individual
frames** — each frame is a separate training sample.

### Extract frames from videos

```bash
# Sample 10 frames per 5-second video clip
python3 -c "
from trainer.prepare_dataset import DatasetPreparer

preparer = DatasetPreparer('/videos')
dataset_info = preparer.prepare_training_dataset(
    run_id='emotionnet_2cls_run01',
    train_fraction=0.7,
    seed=42
)
print(f'Train: {dataset_info[\"train_count\"]} samples')
print(f'Test:  {dataset_info[\"test_count\"]} samples')
print(f'Hash:  {dataset_info[\"dataset_hash\"][:12]}...')
"
```

### Expected directory layout

```
/videos/
├── dataset_all/
│   ├── happy/
│   │   ├── frame_0001.jpg
│   │   └── ...
│   └── sad/
│       ├── frame_0001.jpg
│       └── ...
├── train/          ← generated by DatasetPreparer
├── test/           ← generated by DatasetPreparer
└── manifests/      ← JSONL manifests per run
```

### Convert to TAO format

```bash
# Using the TAO training container
docker exec reachy-tao-train \
  tao emotionnet dataset_convert \
    -e /workspace/specs/emotion_convert.yaml \
    --input_data /workspace/data/source_list.jsonl \
    --output_dir /workspace/data/emotionnet_2cls/processed
```

The `emotion_convert.yaml` spec controls class mapping, train/val split ratio,
and image resizing. See `docs/gpt/2025-10-15-Train EmotionNet for SAD.md` for a
complete walkthrough of the conversion spec.

## Step 2: Fine-Tune with TAO

### Using the training orchestrator (recommended)

```bash
python3 trainer/train_emotionnet.py \
  --config trainer/tao/specs/emotionnet_2cls.yaml \
  --dataset /videos \
  --output /workspace/experiments \
  --mlflow-experiment emotionnet_2cls \
  --seed 42
```

This runs the full pipeline:
1. Dataset preparation (balanced split + JSONL manifests)
2. MLflow run creation with config logging
3. TAO training via Docker (`reachy-tao-train`)
4. Metric parsing (loss, accuracy, F1)
5. Gate A validation (F1 ≥ 0.84 for 2-class)

### Manual TAO training

```bash
docker exec reachy-tao-train \
  tao emotionnet train \
    -e /workspace/specs/emotionnet_2cls.yaml \
    -r /workspace/experiments/emotionnet_2cls_run01 \
    -k tlt_encode
```

### Resuming from a checkpoint

```bash
docker exec reachy-tao-train \
  tao emotionnet train \
    -e /workspace/specs/emotionnet_2cls.yaml \
    -r /workspace/experiments/emotionnet_2cls_run01 \
    -k tlt_encode \
    --resume_model_weights /workspace/experiments/emotionnet_2cls_run01/weights/best.tlt
```

## Step 3: Validate Quality Gates

The training orchestrator enforces two quality gates:

### Gate A — Offline validation

| Metric | 2-class threshold | 6-class threshold |
|--------|-------------------|-------------------|
| F1 macro | ≥ 0.84 | ≥ 0.75 |
| Per-class F1 | ≥ 0.75 | ≥ 0.65 |
| Balanced accuracy | ≥ 0.85 | ≥ 0.78 |

### Gate B — On-device performance (after export)

| Metric | Threshold |
|--------|-----------|
| Latency P50 | ≤ 120 ms |
| Latency P95 | ≤ 250 ms |
| GPU memory | ≤ 2.5 GB |
| F1 macro | ≥ 0.80 (2-cls) / ≥ 0.70 (6-cls) |

Gates are evaluated automatically by `TrainingOrchestrator.validate_gates()` and
logged to MLflow.

## Step 4: Export to TensorRT

```bash
python3 trainer/export_to_trt.py \
  --model /workspace/experiments/emotionnet_2cls_run01/weights/best.tlt \
  --output /workspace/engines \
  --name emotionnet_2cls \
  --precision fp16 \
  --batch-size 1
```

This uses TAO 5.3 (the `reachy-tao-export` container) to produce a `.engine`
file. INT8 export with calibration data is also supported:

```bash
python3 trainer/export_to_trt.py \
  --model /workspace/experiments/emotionnet_2cls_run01/weights/best.tlt \
  --output /workspace/engines \
  --name emotionnet_2cls \
  --precision int8 \
  --calibration-data /workspace/calibration
```

## Step 5: Deploy to Jetson

### Copy the engine to the Jetson

```bash
scp /workspace/engines/emotionnet_2cls_fp16.engine \
  jetson:/opt/reachy/models/emotion.engine
```

### Update the DeepStream SGIE config

The emotion classifier runs as a **secondary GIE** in the DeepStream pipeline:

```ini
[property]
gpu-id=0
process-mode=2                  # secondary
gie-unique-id=3
operate-on-gie-id=1             # run on PGIE faces
onnx-file=/opt/reachy/models/emotionnet_2cls.onnx
model-engine-file=/opt/reachy/models/emotionnet_2cls_fp16.engine
network-mode=2                  # FP16
batch-size=16
labelfile-path=/opt/reachy/models/labels_emotion_2cls.txt
```

The **tracker caching** in DeepStream means EmotionNet does not re-run every
frame — only when a new face appears or the tracked face box changes
substantially. See `docs/gpt/2025-10-15-DeepStream emotion classification.md`
for the complete pipeline wiring.

## Key Configuration Parameters

### 2-Class Config (`emotionnet_2cls.yaml`)

```yaml
model:
  arch: "resnet18"
  input_shape: [224, 224, 3]
  num_classes: 2
  pretrained_weights: "imagenet"
  freeze_blocks: [0, 1]      # freeze early layers
  dropout_rate: 0.3

training:
  batch_size: 32
  num_epochs: 50
  optimizer: "adam"
  learning_rate: 0.001
  lr_schedule:
    type: "cosine"
    warmup_epochs: 5
  loss:
    type: "categorical_crossentropy"
    label_smoothing: 0.1
  class_weights: "balanced"
  mixed_precision: true
```

### Tips for improved results

- **Class balance**: Keep happy:sad close to 1:1 in the training set.
- **Hard negatives**: Include neutral/"tired" faces to sharpen the boundary.
- **Face crops**: Detect/crop/align faces before classification — EmotionNet
  assumes face-centric inputs.
- **Calibration**: Apply temperature scaling on a small real-world set for
  reliable confidence scores.
- **Data volume**: 300–1,000 face crops per class for a baseline; more helps
  with edge cases.

## Toolchain Version Constraints

Per `memory-bank/decisions/004-emotionnet-tao-toolchain.md`:

| Component | Required Version | Container |
|-----------|-----------------|-----------|
| TAO training | 4.x | `reachy-tao-train` |
| TAO export | 5.3 | `reachy-tao-export` |
| DeepStream | 6.x | Jetson runtime |
| TensorRT | 8.6+ | Jetson runtime |

Do **not** use TAO 6.x for training — it is incompatible with the EmotionNet
training flow in this project.

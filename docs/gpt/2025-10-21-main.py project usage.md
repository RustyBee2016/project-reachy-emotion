*   If you browse by **hostname** (`https://aorusai`), the cert should include **aorusai** in SAN, and DNS/hosts should resolve it.

So:

*   If your cert file on disk is actually named `ubuntu1.crt`, keep:

    ssl_certificate     /etc/ssl/certs/ubuntu1.crt;

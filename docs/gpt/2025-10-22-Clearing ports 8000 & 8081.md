**ChatGPT RESPONSE:**

You're right---that ExecStart is exactly why things keep feeling "off."

• That unit is **pinning uvicorn to port 8000** with **\--workers 2**
(production-style), so

whenever the service is enabled/started it grabs 8000.

• Your dev habit is to run **8081** with \--reload from /apps/api. Those
two worlds are

crossing wires.

A couple principles to lock in:

• **Systemd unit = prod-ish**: no \--reload, keep \--workers N, bind to
**8000**, and use a

**stable WorkingDirectory** + correct module path.

• **Dev runs = manual**: \--reload, bind to **8081**, run from
/apps/api.

You've got two clean ways to write the unit---pick one and stick to it.

**Option A: unit runs from repo root**

\[Unit\] Description=Reachy Media Mover API (FastAPI)
After=network-online.target \[Service\] User=rusty_admin
WorkingDirectory=/media/rusty_admin/project_data/reachy_emotion
EnvironmentFile=/media/rusty_admin/project_data/reachy_emotion/apps/api/.env
ExecStart=/media/rusty_admin/project_data/reachy_emotion/apps/api/.venv/bin/uvicorn
\\ \--app-dir . apps.api.app.main:app \--host 127.0.0.1 \--port 8000
\--workers 2 Restart=on-failure \[Install\] WantedBy=multi-user.target

Notes:

• **Don't** use \--reload in the systemd unit (it conflicts with
\--workers and isn't for prod).

• Point EnvironmentFile with an **absolute path** so it works regardless
of

WorkingDirectory.

• \--app-dir . should match the WorkingDirectory so imports are
deterministic.

• Keep the dotted path aligned with your tree: apps.api.app.main:app
(repo root) **or**

app.main:app (api dir).

**What to do now**

1\. Edit the unit:

sudo nano /etc/systemd/system/fastapi-media.service

...updated to **Option A** above.

2\. Apply:

sudo systemctl daemon-reload sudo systemctl disable
fastapi-media.service \# keep it off during dev sudo systemctl stop
fastapi-media.service

3\. For dev runs (from /media/\.../reachy_emotion/apps/api):

source .venv/bin/activate uvicorn \--app-dir . app.main:app \--reload
\--port 8081 \--env-file .env

If later you want the Nginx path live again:

sudo systemctl enable \--now fastapi-media.service

This separation (unit on **8000**, manual dev on **8081**) removes the
port tug-of-war and the import path confusion in one go.

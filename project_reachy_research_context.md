# Project Reachy – Research Context

## Motivation and Background
Project Reachy addresses a critical gap in human-robot interaction: deploying accurate emotion perception systems on resource-constrained edge devices while maintaining strict privacy guarantees and ethical compliance. Traditional cloud-based emotion recognition systems introduce latency, privacy risks, and dependency on external infrastructure, making them unsuitable for companion robotics where real-time responsiveness and user trust are paramount. The system implements a 3-class emotion classifier (happy, sad, neutral) fine-tuned from HSEmotion's EfficientNet-B0 backbone, pre-trained on VGGFace2 and AffectNet datasets totaling over 450,000 facial expression images. This architecture was selected after empirical analysis showed it provides approximately 3× latency improvement (≈40 ms vs. 120 ms budget) and 3× memory headroom (≈0.8 GB vs. 2.5 GB limit) compared to prior ResNet-50 baselines, while maintaining target accuracy through compound scaling and domain-aligned transfer learning. The thermal and GPU margin preserved by EfficientNet-B0 is essential for supporting concurrent gesture planning, LLM-driven empathetic dialogue, and future multimodal features on the Jetson Xavier NX platform without thermal throttling or OOM failures. Privacy-by-design principles enforce local-first data handling with no raw video egress, mTLS/JWT authentication, configurable retention TTLs for temporary media, and explicit user consent mechanisms throughout the data generation, labeling, and continuous improvement loops. These architectural choices directly address ethical concerns raised in recent audits of commercial emotion-recognition deployments, including demographic bias, lack of transparency, and unauthorized data retention. The project further incorporates GDPR-compliant data governance (right to be forgotten, DSAR processes) and WCAG 2.1 accessibility standards to ensure responsible deployment in real-world companion robot scenarios.

## Research Platform
The research platform implements a distributed three-node architecture designed for separation of concerns, fault isolation, and reproducible ML operations. Ubuntu 1 serves as the model host, running PyTorch-based EfficientNet-B0 fine-tuning workflows, the Media Mover FastAPI service (base URL: `https://10.0.4.130/api/media`), PostgreSQL 16 metadata database, and MLflow experiment tracking with file-backed artifact storage. Ubuntu 2 functions as the application gateway, hosting a Streamlit web UI for video generation and labeling, a FastAPI reverse proxy that receives emotion events from the Jetson, routes LLM inference requests, and exposes WebSocket endpoints for gesture cues. The Jetson Xavier NX edge device runs NVIDIA DeepStream SDK 6.x with TensorRT 8.6+ engines, performing real-time emotion classification at 30 FPS with strict performance budgets (p50 latency ≤120 ms, GPU memory ≤2.5 GB) and emitting JSON telemetry without raw video egress. Media flows through a canonical filesystem layout (`/videos/{temp,train,test,thumbs,manifests}`) on ext4 or ZFS, with atomic promotion operations moving accepted clips from `temp/` into `train/<label>/` subdirectories, followed by per-run frame extraction that samples 10 random frames per video into run-scoped datasets (`train/run/<run_id>/<label>/`). Ten specialized agents orchestrated via n8n workflows handle distinct responsibilities: ingest (checksum, metadata extraction), labeling (3-class enforcement, balance tracking), promotion (filesystem mutations, audit logs), reconciliation (drift detection, manifest rebuilds), training (two-phase fine-tuning with frozen/unfrozen stages), evaluation (F1, ECE, Brier metrics), deployment (ONNX→TensorRT conversion, shadow/canary stages), privacy (TTL purges, GDPR compliance), observability (Prometheus metrics, Grafana dashboards), and gesture control (Reachy SDK integration with emotion-conditioned movement). MLflow lineage tracking captures dataset hashes, ZFS snapshot tags, hyperparameters, confusion matrices, and calibration curves for every training run, enabling reproducible experiments and model provenance auditing. The system enforces three deployment gates: Gate A validates offline metrics (F1 ≥0.84, balanced accuracy ≥0.85, ECE ≤0.08), Gate B measures shadow-mode performance on the robot (latency, memory, per-class floors), and Gate C monitors limited user rollout with automatic rollback capabilities if complaint rates exceed 1% of sessions.

## Literature Review & Related Work
The project's technical foundation rests on EfficientNet's compound scaling methodology, which systematically balances network depth, width, and resolution to achieve superior accuracy-efficiency trade-offs compared to manually-designed architectures like ResNet and VGG. Transfer learning from VGGFace2 (3.3M face images across 9,131 identities) and AffectNet (450K facial expression images with 8-class Ekman labels) provides domain-aligned feature representations that significantly outperform generic ImageNet pre-training for facial expression recognition tasks, as demonstrated by HSEmotion's state-of-the-art results on benchmark datasets. The decision to replace NVIDIA TAO's EmotionNet (originally designed for driver monitoring with action unit detection) with HSEmotion EfficientNet-B0 addresses a critical domain mismatch: driver drowsiness features (eye closure, head pose) do not transfer effectively to cinematic emotion expressions (subtle sadness, contemplative neutrality) present in synthetic training videos generated by Luma and Runway APIs. Edge deployment literature emphasizes the importance of DeepStream SDK and TensorRT optimization for achieving real-time performance on Jetson-class devices, with recent work showing that FP16 quantization and layer fusion can reduce inference latency by 3-5× while maintaining accuracy within 1-2% of FP32 baselines. The project extends these findings by demonstrating that careful model selection (EfficientNet-B0 vs. ResNet-50) can provide an additional 3× latency improvement beyond standard TensorRT optimizations, creating essential thermal headroom for concurrent gesture planning and multimodal processing. Human-in-the-loop dataset curation practices, including active learning, label quality auditing, and class balance enforcement, are well-established in affective computing research as critical for mitigating annotation noise and demographic bias in facial expression datasets. The gesture-coupled LLM integration draws from multimodal HRI literature that demonstrates synchronized verbal and non-verbal robot behaviors significantly improve user engagement, trust, and perceived empathy compared to speech-only or gesture-only interactions. Privacy-by-design architectural choices—local-only processing, configurable retention policies, explicit consent mechanisms, and GDPR-compliant data governance—directly respond to ethical concerns raised in recent audits of commercial emotion-recognition systems, including Amazon Rekognition and Microsoft Azure Face API, which documented unauthorized data retention, demographic bias (lower accuracy for darker skin tones and female subjects), and lack of transparency in model decision-making. The three-gate deployment framework (offline validation, shadow mode, limited rollout) aligns with responsible AI deployment practices advocated by Partnership on AI and IEEE standards for algorithmic accountability in high-stakes human-facing systems.

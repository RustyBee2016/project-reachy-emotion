# ADR 006: ResNet-50 with AffectNet + RAF-DB Pre-training

**Status**: Accepted  
**Date**: 2025-11-29  
**Supersedes**: ADR 004 (EmotionNet TAO Toolchain)

---

## Context

The original architecture used NVIDIA TAO EmotionNet for emotion classification. However, EmotionNet was designed for **driver monitoring** (facial action units, drowsiness detection), not human emotion classification. This created a significant domain mismatch:

1. **Pre-training domain**: EmotionNet was trained on driver-facing camera data with specific lighting, angles, and expressions relevant to drowsiness/distraction detection
2. **Feature representation**: The learned features emphasize action units (eye closure, head pose) rather than emotional valence
3. **Label semantics**: The original task (alertness states) doesn't map cleanly to discrete emotions like happy/sad

The synthetic videos generated by Luma/Runway have a cinematic, indoor aesthetic with subtle expressions (e.g., quiet crying, contemplative sadness) that require emotion-specific feature representations.

---

## Decision

Replace EmotionNet with **ResNet-50 pre-trained on AffectNet + RAF-DB datasets** for emotion classification.

### Model Specification
- **Architecture**: ResNet-50 backbone with custom classification head
- **Pre-training**: AffectNet (~450K images) + RAF-DB (~30K images) facial expression datasets
- **Placeholder name**: `resnet50-affectnet-raf-db`
- **Storage path**: `/media/rusty_admin/project_data/ml_models/resnet50`
- **Classes**: Binary (happy, sad) with expansion capability to 8 emotions

### Training Strategy
1. **Phase 1 (epochs 1-5)**: Freeze backbone, train classification head only
2. **Phase 2 (epochs 6+)**: Unfreeze layer4 + fc, fine-tune with 10x lower learning rate for backbone

### Key Techniques
- **Mixed precision (FP16)**: Faster training, reduced memory
- **Mixup augmentation**: Regularization for small datasets
- **Label smoothing (0.1)**: Improved calibration for Gate A ECE requirements
- **Cosine LR schedule with warmup**: Stable convergence

---

## Rationale

| Aspect | EmotionNet | ResNet-50 (AffectNet+RAF-DB) |
|--------|------------|------------------------------|
| Pre-training domain | Driver monitoring | Facial expressions |
| Feature alignment | Action units | Emotional valence |
| Transfer difficulty | High (domain shift) | Low (same domain) |
| Expected accuracy | ~70-75% | ~85-90% |
| Fine-tuning data needed | More | Less |

AffectNet and RAF-DB are the standard benchmarks for facial expression recognition (FER). Models pre-trained on these datasets have learned features specifically for emotional valence, making transfer to our binary classification task much more effective.

---

## Consequences

### Positive
- **Better domain alignment**: Pre-trained features directly relevant to emotion classification
- **Reduced data requirements**: Effective transfer with smaller synthetic dataset
- **Improved calibration**: Label smoothing helps meet Gate A ECE ≤ 0.08 requirement
- **Flexible architecture**: Easy expansion to 8-class or multi-task (emotions + valence/arousal)

### Negative
- **Dependency change**: Requires PyTorch + timm instead of TAO-only workflow
- **Weight acquisition**: Need to obtain or train AffectNet+RAF-DB weights (placeholder for now)
- **Larger model**: ResNet-50 (25M params) vs potential lighter alternatives

### Neutral
- **Deployment unchanged**: Still exports to ONNX → TensorRT for Jetson
- **Quality gates unchanged**: Same Gate A/B/C requirements apply
